### 2. Python 基础语法（1.2 阶段）

**第一部分：基础类型与 None/True/False**
- `None` vs JS `null/undefined` - Python 只有 None，是单例对象
- `is None` vs `== None` - 推荐用 `is` 判断，因为 None 是单例
- `True/False` vs JS `true/false`
- Python Falsy 值：`None, False, 0, "", [], {}`，和js不同的是`[], {}`会视为True真值

**第二部分：Type Hints 类型注解**
- 函数注解：`def func(a: int, b: int) -> int:`
- 变量注解：`age: int = 18`
- Optional 可选类型：`Optional[str]` 或 `str | None`
- Python 3.9+ 支持内置类型：`list[str]` 而非 `List[str]`

**第三部分：列表/字典推导式**
- 列表推导式类似 JS `map + filter` 的结合
- 字典推导式类似 JS `Object.fromEntries()`

### 3. Code Review 收获

**exercise_1 修正：**
- `check_none`: 简化写法 `return "empty" if val is None else val`，这种写法类似js的三元表达式 `true_value if bool else false_value`
- `is_list_empty`: 逻辑修正 `return len(lst) == 0`（空列表返回 True）
- `is_falsy`: 逻辑修正 `return not val`（假值返回 True）

**exercise_2 修正：**
- `add`: 添加返回值类型注解 `-> int`
- `find_index`:
  - 返回类型应该是 `Optional[int]`（找不到返回 None）
  - 循环应该是 `range(len(lst))` 遍历索引，不是直接遍历元素

## 💡 关键收获

1. **类型注解**：Python 是运行时不检查类型，但 IDE 可以提示，配合 `Optional` 使用
2. **列表推导式**：`[x for x in list if condition]` 比 JS `map + filter` 更简洁
3. **`is` vs `==`**：判断 None 用 `is`，因为 None 是单例对象
4. **循环索引**：`for i in lst` 遍历元素，`for i in range(len(lst))` 遍历索引
