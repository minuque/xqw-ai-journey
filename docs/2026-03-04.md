### Exercise 1: 基础类型与 None/True/False

理解了 Python 与 JS 的差异：

- `None` vs JS `null/undefined` - Python 只有 None，是单例对象
- `is None` vs `== None` - 推荐用 `is` 判断，因为 None 是单例
- `True/False` vs JS `true/false` - 布尔值大小写不同
- **Python Falsy 值**：`None, False, 0, "", [], {}`（注意：空列表`[]`和空字典`{}`在 Python 中是 Falsy，而 JS 中它们是 Truthy）

### Exercise 2: Type Hints 类型注解

学习了 Python 类型注解的写法：

- 函数注解：`def func(a: int, b: int) -> int:`
- 变量注解：`age: int = 18`
- Optional 可选类型：`Optional[str]` 或 `str | None`
- Python 3.9+ 支持内置类型：`list[str]` 而非 `List[str]`

### Exercise 3: 列表/字典推导式

掌握了推导式的使用：

- 列表推导式类似 JS `map + filter` 的结合
- 字典推导式类似 JS `Object.fromEntries()`

## 💡 关键收获

1. **类型注解**：Python 是运行时不检查类型，但 IDE 可以提示，配合 `Optional` 使用
2. **列表推导式**：`[x for x in list if condition]` 比 JS `map + filter` 更简洁
3. **`is` vs `==`**：判断 None 用 `is`，因为 None 是单例对象
4. **循环索引**：`for i in lst` 遍历元素，`for i in range(len(lst))` 遍历索引

## 📊 Python vs JS 对比速查表

| 场景 | JavaScript | Python |
|------|------------|--------|
| 空值 | `null`, `undefined` | `None` (单例) |
| 判断空值 | `val === null` | `val is None` |
| 布尔值 | `true`, `false` | `True`, `False` |
| Falsy 值 | `null, undefined, false, 0, NaN, ""` | `None, False, 0, "", [], {}` |
| 类型注解 | `const x: number = 1` (TypeScript) | `x: int = 1` |
| 列表推导 | `arr.map().filter()` | `[x for x in lst if condition]` |
