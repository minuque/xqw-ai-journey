## 📚 今天学了什么

### 环境搭建

- 创建 `src/python-fastapi-foundation/1.advance.py` - Pydantic + 异步编程练习文件（1.3/1.4 阶段）

### Exercise 4: Generics 泛型类型

学习了泛型类型的使用和常见陷阱：

**关键修正**：
- `get_keys`: 需要 `list(d.keys())` 转换，不是直接 `[d.keys()]`
- `get_length`: 参数类型注解 `val: str | list`，不是 `val) -> int:`

### Exercise 5: 元组与可迭代对象

深入理解了元组和可迭代对象的概念：

- **可迭代对象**：能用 for 循环遍历（list, str, tuple, set, dict, range）
- **元组**：不可变的序列，可作为 dict 的 key（dict 字典底层是哈希表）
- **列表**：可变的序列，不可作为 dict 的 key

### Exercise 6: 装饰器模式

掌握了装饰器的原理和应用场景：

**关键修正**：
- `timer`: 需要计算时间差 `end - start`，需要 `return result`
- `cache`:
  - 变量名不要用 `dict`（覆盖内置类型）
  - 用 `if key in cache_dict` 检查，不是 `if dict[key]`
  - key 可用 `frozenset` 兼容 args 和 kwargs
  - 原因是 kwargs 是字典不可哈希

### Exercise 7: dataclass

学习了 `@dataclass` 装饰器的使用：

- 自动生成 `__init__`, `__repr__`, `__eq__`
- 适合简单的数据对象（Model、DTO）
- 不加的话需要手动写构造函数

### Final Exercise: 综合练习

完成了任务管理器综合练习：

**关键修正**：
- 变量名 `nextId` 改为 snake_case: `next_id`
- `complete_task` 需要类型注解 `task_id: int -> bool`
- 为 `add_task` 添加 `@log_call` 日志装饰器

## 💡 关键收获

1. **装饰器应用场景**：日志记录（@log_call）、缓存（@cache）、计时（@timer）
2. **cache 装饰器优化**：用 frozenset 兼容 args 和 kwargs 作为 key
3. **dataclass vs 普通 class**：dataclass 自动生成常用方法，适合数据模型
4. **类型注解**：Python 运行时不检查，但 IDE 提示很有用

## 📊 装饰器原理对比

| 概念 | JavaScript (装饰器提案) | Python |
|------|------------------------|--------|
| 基础装饰器 | `@decorator` | `@decorator` |
| 带参数装饰器 | `@decorator(arg)` | `@decorator(arg)` |
| 工厂模式 | `function decorator(arg) { return (target) => {} }` | `def decorator(arg): return decorator_func` |
| 缓存装饰器 | 用 Map 存储 | 用 dict 存储 |
| 日志装饰器 | 用 console.log | 用 print/logging |
